# SDG 3 Indicator Text Classification

Multi-label text classification system for identifying relevant (Sustainable Development Goal) indicators in development and humanitarian text. 

**Evaluation metric:** Hamming Loss (lower = better)  
**Best result:** See `outputs/all_experiments_final.csv`

## Setup Instructions

### Step 1 — Get the datasets

Download both CSV files from the course assignment page:

- `Devex_train.csv`
- `Devex_test_questions.csv`

Upload them to your Google Drive at the following path:

```
/MyDrive/SDG/Devex_train.csv
/MyDrive/SDG/Devex_test_questions.csv
```

> **Note:** The data files are excluded from this repository via `.gitignore` due to size. You must upload them manually before running any notebook.

### Step 2 — Open in Google Colab

Each notebook mounts Google Drive automatically. Open any notebook in Colab via:

```
File → Open notebook → GitHub → https://github.com/Muen1/SDG_Indicator_Text_Classification.git
```

Or upload notebooks directly to Colab and mount Drive when prompted.

### Step 3 — Install dependencies

Each notebook installs its own dependencies in the first cell. You can also install everything at once:

```bash
pip install -r requirements.txt
```

## How to Reproduce Results

**Run notebooks in order.** Each notebook saves artifacts to Google Drive that the next notebook reads.

```
01 → 02 → 03 → 04
```

| Notebook | Reads from Drive | Writes to Drive |
|---|---|---|
| 01 — Data Pipeline | `Devex_train.csv`, `Devex_test_questions.csv` | `features/`, `train_processed.csv` |
| 02 — Modeling | `features/` | `models/`, `all_experiment_results.csv` |
| 03 — Feature Tuning | `features/`, `models/` | `all_experiments_final.csv`, `visualizations/` |
| 04 — Results | `models/`, `all_experiments_final.csv` | `outputs/test_predictions.csv`, `visualizations/` |

> All intermediate files are saved to `/content/drive/MyDrive/SDG/` inside Colab. Do not change the `PROJECT_FOLDER` path unless you update it consistently across all notebooks.

## Experiments Summary

12 experiments across Notebooks 02 and 03, tracked by Hamming Loss:

| # | Experiment | Key Change |
|---|---|---|
| 1 | Baseline LR | OneVsRest + Logistic Regression, default params |
| 2 | Random Forest | 100 trees, OvR wrapper |
| 3 | Linear SVM | LinearSVC, OvR wrapper |
| 4 | Tuned LR | Grid search over C, refitted on full training data |
| 5 | Word2Vec + LR | In-corpus 100-dim averaged word vectors |
| 6 | max_features sweep | TF-IDF vocab size: 1k / 3k / 5k / 7k / 10k |
| 7 | n-gram range | Unigram vs bigram vs trigram |
| 8 | Class balancing | LR with `class_weight='balanced'` |
| 9 | CountVectorizer vs TF-IDF | Raw counts vs IDF-weighted (bug-fixed label alignment) |
| 10 | Stemming vs Lemmatization | Porter stemming vs WordNet lemmatization |
| 11 | GloVe + LR | Pre-trained 100-dim GloVe (6B tokens) |
| 12 | Threshold tuning | Calibrated SVM, sweep 0.10–0.60 cut-off |

Full results with all metrics: `outputs/all_experiments_final.csv`

## Notes on Reproducibility

- All notebooks use `RANDOM_SEED = 42` for all random operations.
- The train/validation split (80/20) uses `random_state=42` and is reproduced identically across all notebooks by applying the same parameters to the same data.
- GloVe download in Notebook 03 requires an internet connection (~822 MB). It is cached in `/tmp/glove/` for the duration of the Colab session.
- Runtime estimates on a free Colab CPU instance: Notebook 01 ~8 min, Notebook 02 ~12 min, Notebook 03 ~20 min (GloVe download included), Notebook 04 ~5 min.
